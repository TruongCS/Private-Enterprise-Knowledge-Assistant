# Private-Enterprise-Knowledge-Assistant

This project builds a financial Q&A agent from an annual report. It uses Docling to convert a PDF report into Markdown, stores extracted tables in SQLite, stores narrative text chunks in a FAISS vectorstore, and uses a LangChain agent to answer financial questions.

The system has two retrieval paths:

1. **Vectorstore retrieval**
   - Source: Markdown report text
   - Storage: FAISS vectorstore
   - Purpose: narrative questions, business descriptions, risks, strategy, qualitative explanations

2. **SQLite table retrieval**
   - Source: Markdown tables extracted from the report
   - Storage: `uber_financials.db`
   - Purpose: exact financial numbers, revenue, net income, EBITDA, cash flow, segment data

The agent decides which tool to use depending on the question.

## Files

- `Private_Enterprise_Knowledge_Assistant.ipynb`  
  Main Google Colab notebook. Runs PDF conversion, ingestion, vectorstore creation, SQLite extraction, and agent setup.

- `Uber's_10-K_annual_report.pdf`  
  Original annual report PDF.

- `uber_report.md`  
  Markdown version of the PDF generated by Docling.

- `uber_financials.db`  
  SQLite database containing tables extracted from the Markdown report.

- `vectorstore/`  
  FAISS vectorstore containing embedded text chunks from the Markdown report.
